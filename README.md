# RHYTHMICA
BeatorajaとLR2の仕様を可能な限り融合したBMSプレイヤー  
**警告: C++初学者でありプログラミング学ぶために作っている部分があるので進行の仕方が変な可能性があります。**

## 現在やっていること
BMSファイルを読んで中身を出力するプログラムを作成中

### LR2から持ってくる大きな仕様 
- 負数STOP(基本的に値は-1)で1/192小節スキップするバグ(これを仕様にする)
- 曲開始時以外でBPMが1未満になるとBPM欄がスロットのごとく変化し続けるバグ(仕様にする)(その後にBPMが1以上が指定されたら元に戻るようにする)
- 負数BPMの地点を過ぎたときノーツが逆行する仕様
- BGMレーンから一定程度レーンに流れてくるノーツを増やす機能(EXTRA MODE)
- データベースの更新のタイプ「開いたフォルダに対して自動で行う」を実装
- 差分名(#SUBTITLE)でソート可能にする(難度フィルタ)
- スキン選択画面でのスキンプレビュー機能(内蔵のオートプレイ用譜面を仕込み、再生する)
- lr2crsファイルの読み込み対応
- LR2スキンをプレイスキンに限って対応
- LR2特有のIR仕様
- LR2の難易度表フォーマット対応
- LR2にしかないゲージの実装(P.ATTACK, G.ATTACKなど)
- オーディオエフェクトの実装(ライセンス周り頑張れたらVST対応できると強い)
- BATTLEモードの実装(片方だけオートプレイにできるようにもする)
-BPMが指数などを使って表記されている場合にも正しく読めるようにする(1.2e2のように)

### Beatorajaから持ってくる大きな仕様
- SCROLL定義の対応(SCROLL出現消失バグもなんとかして仕様として実装)
- BGAのために大体の映像コーデックに対応させる
- VOLWAV定義も一応対応する
- 不可視ノーツ表示機能
- 練習モード実装
- IIDXコントローラーなどのアナログスクラッチとデジタルスクラッチ両方の規格に対応させる
- 次世代難易度表フォーマット対応
- ランセレ
- GASの実装(SELECT TO UNDERのみで良い)
- 判定アルゴリズム3種(コンボ優先、スコア優先、最下ノーツ優先)の実装
- 曲開始前のローディング画面でSTARTやSELECTで譜面プレビューを表示する
- DRP (Discord Rich Presence) 対応
- スクショクリップボードにコピー
- Beatorajaにしかないゲージの実装(アシストEASY、EX-HARDなど)
- Beatorajaスキンをプレイスキンに限って対応
- SUDDEN+、HIDDEN+、LIFTの実装
- 緑数字の表示(ソフランがある場合は最大値と最小値の表示)、白数字の表示
- ノーツ表示時間固定(IIDXで言うFHS)の実装(MAX BPM、MIN BPM、MAIN BPM、START BPM、IIDX FHSなど)
- VERY EASY判定の実装
- bmson対応
- 62進数BMS対応
- IRからフォルダを持ってくる機能
- オートプレイ
- Beatorajaに存在する外部ツール「Songdata-Updater」を参考にしてEverythingを利用した爆速データベース更新を実現する。SQLiteも活用はする。
- !req命令が使えるようにする
- スキン読み込み高速化ファイルの作成をして高速化

### 融合させて強化する機能
- オンラインバトル機能の実装(主要難易度表から選曲して対戦するモード、選曲制限をしないモード、ユーザーによるカスタムルームを作成できるモード、多人数でスコアなど競い合うモード)
- EASYやNORMALゲージなどLR2、Beatoraja、IIDXに共通して存在するゲージはどの仕様を用いるか選択できるようにするし、どれを選んでも正しくスコアとして認める
- データベースがどれだけ用意できるか次第だが、MochaIRなどとうまく連携できればリコメンドツールを内蔵する(自IRが発展したらそこベースで求めても良い)
- ROKINA(Kuroki-Narumi)氏が提唱するBMSの識別(ハッシュ化)における提言を元に、譜面内容が同一であるのに中の文字列の順序が違うだけでハッシュ割れが起きる現象を低減させる
- IRにおいて申告制で譜面内容が合致するが他の情報が異なることでハッシュ割れが起きているBMSを統合する機能
- #RANDOM、#SETRANDOM、#IFの入れ子の制御構造に対応する
- DECIDEは重い必要がないのでデフォルトは超軽量なスキンにしておく(もちろん従来みたいなかっちりしたやつも作れるようにする)
- RESULTは機能はもりもりにするがこれも軽量に振る
- 自動アップデート機能(無効化も可能にする)
- クロスプラットフォームは対応したい。何ならAndroidも対応させたい。Appleは嫌いなのでMacOSもiPhoneやiPadのOSも一切対応する気はない。BeatorajaだとLinuxのとき\がキーコンフィグとしてアサインできない現象はないようにしたい。
- 軽量はとことん突き詰めるがよりモダンなデザインを実装できるようにもする。スキンはもはや伝統工芸品レベルで作者が少ないのでより敷居を下げる規格を作る。あるいは無知からでも制作が完了できるだけのガイドを提示する。
- コースモードの強化 自分で作ったコースを段位認定レベルで設定可能にしてIRとかで他人と共有できるようにしたい
- IIDXの機能で持ってこれる機能は持ってくる(怒られない程度に)詳細オプションとかの設定はIIDX風にすると良さそう
- 内部に触れる人たちのためにターミナルからデバッグ等できるようにする。
- Githubでオープンソース化し様々な人から協力をもらいたい
